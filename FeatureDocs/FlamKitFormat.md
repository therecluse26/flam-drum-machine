# FlamKit Format Specification (flamkit.yaml)

**Status:** v1.0 Core Format
**Priority:** Critical
**File Format:** YAML (human-readable, version-controllable)

---

## Overview

The `flamkit.yaml` file is the central metadata file for a FlamKit drum kit. It defines:
- Kit-level metadata (name, author, license)
- **Microphone channel names** (critical for mixer labeling)
- Drum pieces and their MIDI note assignments
- Velocity layers and round-robin samples for each piece
- Choke groups and articulation definitions

This file is generated by **FlamForge** during kit recording, or hand-authored for custom kits.

---

## File Structure

### Complete Example

```yaml

author: "John Drummer"
license: "CC-BY-4.0"
description: "Professional 10-piece rock kit recorded at Abbey Road Studio 2"
sampleRate: 44100
bitDepth: 24

# Microphone Channel Definitions
# CRITICAL: These names are used to label mixer channels in FlamKit
# Order must match channel order in WAV files (0-indexed)
channels:
  - name: "Kick Close"
    index: 0
  - name: "Kick Sub"
    index: 1
  - name: "Snare Top"
    index: 2
  - name: "Snare Bottom"

    index: 6

  - name: "Room-R"
    index: 8

# Choke Groups (mutually exclusive sounds)
chokeGroups:
  - id: 0
    name: "Hi-Hat"
    pieces: ["HiHat-Closed", "HiHat-Open"]

pieces:
  - name: "Kick"
    midiNote: 36
```

## Field Definitions

### Kit-Level Metadata

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | String | Yes | Human-readable kit name |
| `version` | String | Yes | Semantic version (e.g., "1.2.0") |
| `author` | String | No | Kit creator name |
| `license` | String | No | License (e.g., "CC-BY-4.0", "GPL-3.0") |
| `description` | String | No | Kit description |
| `sampleRate` | Integer | Yes | Sample rate of WAV files (44100 or 48000) |
| `bitDepth` | Integer | No | Bit depth (16, 24, or 32) |

---

### Microphone Channels Section

**Purpose**: Define the name and index of each microphone channel in the multi-channel WAV files.

**Critical Role**: These names are displayed in the FlamKit mixer UI as channel strip labels.

```yaml
channels:
  - name: "Kick Close"
    index: 0
  - name: "Kick Sub"
    index: 1
  - name: "OH-L"
    index: 2
  # ... etc
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | String | Yes | Microphone channel name (displayed in mixer) |
| `index` | Integer | Yes | 0-based channel index in WAV files |

**Parsing Rules**:
- Number of `channels` entries must match number of channels in WAV files
- `index` must be sequential (0, 1, 2, ...)
- `name` should be short (max 16 characters for UI display)
- Common naming conventions:
  - "Kick Close", "Kick Sub"
  - "Snare Top", "Snare Bottom"
  - "OH-L", "OH-R" (overheads)
  - "Room-L", "Room-R", "Ambient"

**Example Multi-Channel Configurations**:

**Minimal 2-Channel (Stereo)**:
```yaml
channels:
  - name: "L"
    index: 0
  - name: "R"
    index: 1
```

**Standard 8-Channel**:
```yaml
channels:
  - name: "Kick Close"
    index: 0
  - name: "Snare Top"
    index: 1
  - name: "Hi-Hat"
    index: 2
  - name: "OH-L"
    index: 3
  - name: "OH-R"
    index: 4
  - name: "Room-L"
    index: 5
  - name: "Room-R"
    index: 6
  - name: "Ambient"
    index: 7
```

**Advanced 16-Channel**:
```yaml
channels:
  - name: "Kick In"
    index: 0
  - name: "Kick Out"
    index: 1
  - name: "Kick Sub"
    index: 2
  - name: "Snare Top"
    index: 3
  - name: "Snare Bottom"
    index: 4
  - name: "HH Close"
    index: 5
  - name: "Tom 1"
    index: 6
  - name: "Tom 2"
    index: 7
  - name: "Tom 3"
    index: 8
  - name: "OH-L"
    index: 9
  - name: "OH-R"
    index: 10
  - name: "Room-L"
    index: 11
  - name: "Room-R"
    index: 12
  - name: "Ambient-L"
    index: 13
  - name: "Ambient-R"
    index: 14
  - name: "Crush"
    index: 15
```

### Choke Groups Section

**Purpose**: Define mutually exclusive drum pieces (e.g., open/closed hi-hat).

```yaml
chokeGroups:
  - id: 0
    name: "Hi-Hat"
    pieces: ["HiHat-Closed", "HiHat-Open"]
  - id: 1
    name: "Cymbals"
    pieces: ["Crash-Choked", "Crash-Open"]
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `id` | Integer | Yes | Unique choke group ID (0-15) |
| `name` | String | Yes | Human-readable group name |
| `pieces` | Array[String] | Yes | List of piece names in this group |

**Behavior**: When any piece in a choke group triggers, all other pieces in the group are immediately stopped.

---

### Drum Pieces Section

**Purpose**: Define each drum piece, its MIDI note, and sample layers.

```yaml
pieces:
  - name: "Kick"
    midiNote: 36
    chokeGroup: null
    velocityLayers:
      - velocityRange: [0, 40]
        roundRobins:
          - file: "Samples/Kick/kick_v020_rr0.wav"
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | String | Yes | Piece name (must match choke group references) |
| `midiNote` | Integer | Yes | MIDI note number (0-127, typically 35-81 for drums) |
| `chokeGroup` | Integer or null | No | Choke group ID (null if not in a group) |
| `velocityLayers` | Array | Yes | List of velocity layers |

---

### Velocity Layers

```yaml
velocityLayers:
  - velocityRange: [0, 40]
    roundRobins:
      - file: "Samples/Kick/kick_v020_rr0.wav"
      - file: "Samples/Kick/kick_v020_rr1.wav"
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `velocityRange` | Array[Int, Int] | Yes | [min, max] MIDI velocity (0-127) |
| `roundRobins` | Array | Yes | List of sample files for this layer |

**Parsing Rules**:
- `velocityRange` must not overlap with other layers
- `velocityRange` should cover full dynamic range (0-127)
- `roundRobins` array can have 1-unlimited samples

---

### Sample File Paths

```yaml
- file: "Samples/Kick/kick_v020_rr0.wav"
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `file` | String | Yes | Relative path to WAV file (from `flamkit.yaml` directory) |

**Rules**:
- Paths are relative to the directory containing `flamkit.yaml`
- Use forward slashes `/` (cross-platform compatible)
- WAV files must have same number of channels as defined in `channels` section
- WAV files must match `sampleRate` specified in kit metadata

## Parsing Implementation

### C++ Parsing (JUCE)

```cpp
// Source/Formats/FlamKitLoader.h
class FlamKitLoader
{
public:
    struct MicrophoneChannel
    {
        juce::String name;
        int index;
    };

    struct KitMetadata
    {
        juce::String name;
        juce::String version;
        juce::String author;
        juce::String license;
        int sampleRate;
        int bitDepth;

        std::vector<MicrophoneChannel> channels;  // CRITICAL: Parsed from YAML

        // ... pieces, choke groups, etc.
    };

    static KitMetadata loadFromFile(const juce::File& yamlFile);

private:
    static void parseChannels(const juce::var& yamlData, KitMetadata& metadata);
};
```

### Parsing Microphone Channels

```cpp
void FlamKitLoader::parseChannels(const juce::var& yamlData, KitMetadata& metadata)
{
    const auto& channelsArray = yamlData["channels"];

    if (!channelsArray.isArray())
    {
        jassertfalse;  // Invalid format
        return;
    }

    for (const auto& channelEntry : *channelsArray.getArray())
    {
        MicrophoneChannel mic;
        mic.name = channelEntry["name"].toString();
        mic.index = static_cast<int>(channelEntry["index"]);

        metadata.channels.push_back(mic);
    }

    // Validate: indices must be sequential 0, 1, 2, ...
    for (size_t i = 0; i < metadata.channels.size(); ++i)
    {
        jassert(metadata.channels[i].index == static_cast<int>(i));
    }
}
```

## Usage in FlamKit

### 1. Kit Loading

```cpp
// In FlamAudioProcessor::loadKit()
KitMetadata metadata = FlamKitLoader::loadFromFile(kitFile);

// Configure multi-channel mixer
perChannelMixer->setNumChannels(
    metadata.channels.size(),
    getMicrophoneNames(metadata)  // Extract names from channels
);

// Configure multi-output buses
std::vector<juce::String> busNames;
for (const auto& channel : metadata.channels)
{
    busNames.push_back(channel.name);
}

multiChannelRouter->configureBuses(busNames);
```

### 2. Mixer UI Labeling

```cpp
// In MixerPanel::onKitLoaded()
void MixerPanel::onKitLoaded(const KitMetadata& metadata)
{
    channelStrips.clear();

    for (const auto& micChannel : metadata.channels)
    {
        auto* strip = new ChannelStripComponent(mixer, micChannel.index);
        strip->setChannelName(micChannel.name);  // "Kick Close", "OH-L", etc.
        channelStrips.add(strip);
    }

    resized();  // Layout fader strips
}
```

## Validation Rules

When parsing `flamkit.yaml`, validate:

1. **Required fields present**: `name`, `version`, `sampleRate`, `channels`, `pieces`
2. **Channel indices sequential**: 0, 1, 2, ..., N-1
3. **WAV files exist** at specified paths
4. **WAV channel count** matches `channels.size()`
5. **WAV sample rate** matches `sampleRate`
6. **Velocity ranges** don't overlap within a piece
7. **Choke group references** valid (piece names exist)
8. **MIDI notes unique** per piece (no duplicates)

## Error Handling

```cpp
KitMetadata FlamKitLoader::loadFromFile(const juce::File& yamlFile)
{
    if (!yamlFile.existsAsFile())
        throw std::runtime_error("Kit file not found: " + yamlFile.getFullPathName().toStdString());

    juce::String yamlContent = yamlFile.loadFileAsString();
    juce::var parsedYaml = parseYAML(yamlContent);  // Use yaml-cpp or similar

    KitMetadata metadata;

    // Parse required fields
    if (!parsedYaml.hasProperty("name"))
        throw std::runtime_error("Missing required field: name");

    metadata.name = parsedYaml["name"].toString();

    // Parse channels
    if (!parsedYaml.hasProperty("channels"))
        throw std::runtime_error("Missing required field: channels");

    parseChannels(parsedYaml, metadata);

    // Validate channel count matches WAV files
    validateSampleFiles(metadata, yamlFile.getParentDirectory());

    return metadata;
}
```

## Summary

The `flamkit.yaml` format is:

✅ **Human-readable**: Engineers can edit in any text editor
✅ **Version-controllable**: Works with Git/SVN
✅ **Microphone-centric**: Channels section defines mixer layout
✅ **Comprehensive**: Supports unlimited velocity layers, round-robins, choke groups
✅ **Validated**: Parser enforces consistency between YAML and WAV files

**The `channels` section is CRITICAL** - it directly controls how the FlamKit mixer UI is generated and labeled.
