Install · Quick start · Build from source · Your first plugin

Install

  1. Download the build for your OS and plugin format (VST3, AU, AAX, or Standalone).
  2. Run the installer, or copy the plugin into your system plugin folder (see the table on the Download page).
  3. Open your DAW and rescan plugins. FlamKit appears under FLAM in your instrument list.
  4. Add FlamKit to an instrument track, then load a kit.

macOS first launch: if Gatekeeper blocks the standalone app, right-click it and choose Open once. Windows: SmartScreen may warn on first run of the installer — choose More info → Run anyway. Both builds are signed; these prompts are normal for new publishers.

Quick start — sound in 5 minutes

  1. Load FlamKit on an instrument track (or launch the standalone app).
  2. Open the kit browser and load a starter kit from the Samples page.
  3. Play your MIDI controller, or drag a MIDI clip onto the track. Velocity selects the layer; round-robins keep repeated hits natural.
  4. Multi-output: in your DAW's I/O settings, route each FlamKit output bus (kick close, snare top, overheads, room…) to its own track. Now mix the kit like a real multitrack session.
  5. No multi-output host? Use the built-in mixer — per-channel volume, pan, solo and mute — and the stereo Main Mix bus.

Build from source

FlamKit is a standard C++20 / JUCE 8 project driven by CMake. The JUCE template generates every plugin format from one definition.

Prerequisites

  • CMake 3.22+ and a C++20 compiler (Clang, MSVC, or GCC)
  • JUCE 8 (fetched by CMake, or point to a local copy)
  • Platform toolchain: Xcode (macOS), Visual Studio 2022 (Windows), or build-essential (Linux)

Clone & build

git clone https://github.com/therecluse26/flam-drum-machine.git
cd flam-drum-machine
mkdir build && cd build
cmake ..
cmake --build . --config Release

Built artifacts land in the build tree under each format's output folder (VST3, AU, AAX, Standalone). Copy them into your plugin folder and rescan.

The build steps here mirror the repository README, which is the single source of truth. If anything drifts, the README wins — please open an issue.

Your first plugin — ship something in an afternoon

FlamKit is designed to be a readable reference for shipping your own JUCE-based sampler. A good first modification:

  • Read the architecture: start with ARCHITECTURE.md — it maps the engine, voice manager, format parser, DSP and plugin entry points.
  • Make one real change: add a new parameter and wire it through the APVTS to the engine, or load a custom flamkit.yaml kit you authored by hand.
  • Build, load in your DAW, hear the difference. That round trip — clone to audible change — is the "afternoon" promise.
  • Open a PR or share it on Discord. Contributions are how the project gets better.

Detailed contributor guidance lives in CONTRIBUTING.md.
